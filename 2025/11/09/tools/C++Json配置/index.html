VSCode C++ Json配置

c_cpp_properties.json

基本结构（可以复制这个然后再适配项目）：

configurations: 一个数组，里面可以存多套配置

name: 配置名称，你可以在命令面板中切换当前IntelliSense配置，常见取值为 Win32，Linux 等

includePath: 包含的路径，告诉IntelliSense去哪里找头文件，通常在引入第三方库后添加对应的路径

defines: 模拟增加宏定义

compilerPath: 指定的模拟的编译器路径（这个很重要，会影响代码补全、语法检查等IntelliSense核心功能）

cStandard和cppStandard: 指定IntelliSense使用的C/C++语言标准，C的可选值为c89，c90，c99，c11，c17，gnu11，C++的可选值为c++98，c++11，c++14，c++17，c++20，c++23，gnu++17。

intelliSenseMode: 指定VSCode模拟的系统+编译器架构，用于决定宏定义与默认头路径，例如在x64 Linux上通常为 linux-gcc-x64 和 linux-clang-x64。

browse: 为项目构建符号数据库，保存了项目中类、函数、变量、宏等符号的位置与关系，供编辑器做全局导航功能。

其中，path 定义需要扫描的路径，主要包括项目源代码、头文件、第三方库和系统库。

⚠️ 不要引入过多的外部路径，会影响数据库的效率

limitSymbolsToIncludedHeaders 控制是否只索引被实际包含的头文件，为布尔值。

databaseFilename 为符号数据库的存储路径，默认在 .vscode/browse.vc.db。

在大型项目中，vscode索引速度慢，可以通过配置browse来减轻压力，在默认情况下，vscode会扫描所有的目录，例如.git，build，外部库，中间文件等。我们可以在 browse.path 中只存放真正的源码路径，不放构建路径。

compilerArgs: 模拟编译器命令行选项

configurationProvider: 如果你使用CMake、Makefile、Meson等扩展，这项可让vscode自动从构建系统读取配置。

version: 文件格式版本。目前最新版本一般是4，不建议更改。

settings.json

这个json可以修改大多数编辑器外观相关的设置，虽然这个主题只讨论C++相关，但还是包括很多相关设置：

  • 基础编辑器行为
  • C++专属配置块
  • IntelliSense/编译配置
  • 代码分析/诊断
  • 格式化（clang-format）
  • 智能提示/补全
  • 调试/运行体验
  • 代码风格偏好（可选）
  • IntelliSense额外配置
