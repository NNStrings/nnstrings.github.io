llvm-12.0.0源码解析-基础数据结构-IntrusiveRefCntPtr

源码详见 llvm/include/llvm/ADT/IntrusiveRefCntPtr.h

RefCountedBase & ThreadSafeRefCountedBase 类

要了解 IntrusiveRefCntPtr，必须先了解 RefCountedBase 类，这是一个为类型添加引用计数的 CRTP mixin 类。它分为线程非安全和线程安全两个版本，分别为 RefCountedBase 和 ThreadSafeRefCountedBase。

classDiagram
    class RefCountedBase~Derived~ {
        <<template>>
        -RefCount: mutable unsigned
        +Retain() void
        +Release() void
    }
    class ThreadSafeRefCountedBase {
        <<template>>
        -RefCount: mutable std::atomic~int~
        +Retain() void
        +Release() void
    }

其中，Retain() 使计数器加一，Release() 使计数器减一，如果计算之后计数器 RefCount 为 0，则释放 Derived 指针。

对于线程安全版本，在处理计数器增减做了线程安全处理。其中，加法不需要考虑被重排到其他位置的问题，只需要保证它是原子加法即可，因为 Retain 本身就意味着一定有一个线程持有该对象。而对于减法，我们要保证析构线程的前后都必须是可见的，不能出现内存释放后有对数据进行操作的行为。

❓ 有没有一种情况，线程A执行了 Retain，线程B执行了 Release，结果问题发生了，B率先释放了内存，A在对计数器进行加1操作出现内存泄漏？
💡 这是对于这个结构的未定义行为，不允许这么使用，而不是这个结构设计的本身有问题

IntrusiveRefCntPtr 类

IntrusiveRefCntPtr 类是一个侵入式智能指针。

IntrusiveRefCntPtr 类是一个模板类，它的模板参数类至少需要包含 Retain 和 Release 这两个方法，一般情况下，这个类需要递归继承自 RefCountedBase 类或 ThreadSafeRefCountedBase 类。

IntrusiveRefCntPtr 不仅支持本身的普通指针、拷贝、移动构造，还支持跨类型的拷贝、移动

大家可能注意到在跨类型的移动中，类直接访问了其他对象的私有成员 S.Obj = nullptr;，IntrusiveRefCntPtr 类通过自己当作了自己的友元来解决这个问题。

在 IntrusiveRefCntPtr 类中，核心函数 Retain 和 Release 存储在 IntrusiveRefCntPtrInfo 类中，以组合的方式存在。

🔑 这种将 Retain 和 Release 独立在外面有很多好处：

  • 将这两个接口函数从智能指针中独立出来，完成解耦
  • 我们可以通过特化 IntrusiveRefCntPtrInfo 来自定义两个接口函数的行为。
  • 可以解决循环头文件包含的问题。（❓没构造出来）

IntrusiveRefCntPtr 以及相关类之间的关系：

classDiagram
    class RefCountedBase~T~ {
        <<abstract>>
        -int refCount
        +Retain() void
        +Release() void
    }

    class ThreadSafeRefCountedBase~T~ {
        <<abstract>>
        -atomic_int refCount
        +Retain() void
        +Release() void
    }

    class IntrusiveRefCntPtrInfo~T~ {
        <<template>>
        -T* ptr
        +IntrusiveRefCntPtrInfo(T*)
        +Retain() void
        +Release() void
    }

    class IntrusiveRefCntPtr~T~ {
        <<template>>
        -IntrusiveRefCntPtrInfo~T~ info
        +IntrusiveRefCntPtr(T*)
        +~IntrusiveRefCntPtr()
        +swap(IntrusiveRefCntPtr&) void
        +reset() void
        +resetWithoutRelease() void
    }

    class T {
        <<T必须递归继承自RefCountedBase或ThreadSafeRefCountedBase>>
    }

    %% 继承关系
    T --|> RefCountedBase~T~ : CRTP
    T --|> ThreadSafeRefCountedBase~T~ : CRTP

    %% 组合关系
    IntrusiveRefCntPtr~T~ *-- IntrusiveRefCntPtrInfo~T~ : 包含

    %% 依赖关系
    IntrusiveRefCntPtr~T~ ..> T : 模板参数

    %% 模板参数关系
    note for T "模板参数T必须满足：
    T : public RefCountedBase< T > 或 public ThreadSafeRefCountedBase< T >"
